## vitepress

这是一个静态站点生成项目([网站地址](https://vitepress.dev/))，核心就是文件路由系统和对markdown 的良好支持，可以将md文件快速转换为html文件，并配置美观的页面。接下来讲一下核心的几个点：

- 项目结构
- 路由
- 配置项

### 2. 项目结构

可以看出vitepress项目结构很简单，就是一个默认工作目录，这个docs是我们创建时指定的，核心就是docs/.vitepress下的东西，其中cache中包含着开发服务器，dist则是构建的产物，开发服务器默认是从index.md来访问整个项目的

```shell
.
├─ docs
│  ├─ .vitepress
|  |  └─ dist
|  |  └─ cache
│  │  └─ config.js
│  ├─ api-examples.md
│  ├─ markdown-examples.md
│  └─ index.md
└─ package.json
```

### 2. 文件路由

因为这个项目主要是文档类的静态网站，所以其使用了文件路由这一特性，系统中的文件，源文件自动会被映射成url，例如：`docs/test/hello.md -> https://host/docs/test/hello.html`这是一种对文件十分高效的组织方式
当然这个项目还支持如动态路由，前缀修改等方式。

接下来需要介绍几个关键概念

#### 项目根目录

一般是docs的上层目录

#### 源目录

markdown文件所在的位置，可以通过`srcDir`来指定

#### base
网站会自动为所有的path添加的前缀，例如:`docs/test/hello -> https://host/path...`

#### 动态路由

这个一般涉及参数传递等操作，markdown文件是一个纯静态的页面，如果我们想根据不同的情况返回不同的页面就需要路径加载文件`[params].paths.js/ts`

...这个东西在纯静态的页面确实用的不是很多

## github actions

这是github提供的持续集成功能，融合在github的开发工作流中，提供项目的集成，管理，构建和部署等功能。

### 基本使用

使用十分方便，我们就拿之前的vitepress举例，可以很容易地将静态网站项目托管到github上。

1. 配置`github/workflows/deploy.yml`这里可以直接参考`https://vitepress.dev/zh/guide/deploy#github-pages`
2. 在project -> setting -> pages -> build and deploy 选择GitHub actions
3. 本地git push代码到平台上即可
