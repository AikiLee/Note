二者最主要的区别就是工作的层级不同， 在osi网络模型中，系统代理工作在第7层，处理使用了标准系统的应用，而tun工作在网络层，管理所有的数据包，所以可以所有的应用程序进行管理。


| 特性   | 系统代理                                        | tun                                                                                           |
| ---- | ------------------------------------------- | --------------------------------------------------------------------------------------------- |
| 实现原理 | 是一个中间服务器，专门为支持代理功能的应用程序（如网页浏览器、下载工具等）转发网络请求 | 是一种虚拟网络接口，它在操作系统内核中创建，表现得就像一块真实的物理网卡（如你的以太网卡或 Wi-Fi网卡）。<br>它不处理 HTTP 这样的上层协议，而是直接处理原始的 IP数据包。 |
| 工作层级 | osi 7                                       | osi3                                                                                          |
| 工作协议 | 特定协议：http/https，socket                      | 基于ip的所有协议都可以被管理，直接作用于tcp/udp                                                                  |
| 优点   | 效率高                                         | 可以作用全局设备                                                                                      |
| 缺点   | 仅限于特定协议                                     | 复杂度较高                                                                                         |


在具体实现上：

系统代理它是一个中间服务器，专门为支持代理功能的应用程序（如网页浏览器、下载工具等）转发网络请求。

而tun是一种虚拟网络接口，它在操作系统内核中创建，表现得就像一块真实的物理网卡（如你的以太网卡或 Wi-Fi网卡）。
它不处理 HTTP 这样的上层协议，而是直接处理原始的 IP数据包。
